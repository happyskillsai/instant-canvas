# Gotchas

Lessons learned the hard way so we don't repeat them.

- [Runtime (kernel & CLI)](gotchas/runtime.md) — stdout flush truncation, keep-alive poll races, same-version kernel staleness, health-ping liveness, `/tmp` symlinks, re-serializing a file to add one field (and why a *nested* field needs a JSON scanner rather than a regex), a reader-facing write that would have changed what a canvas *is*, three separately-correct forgiving layers (a config that swallows parse errors, a resolver that drops junk, a watcher that skips dotfiles) composing into a feature that silently did not exist, a validator error becoming a wall of red in the reader's browser, a refused file leaking its own first bytes through `JSON.parse`, and the lessons the removed folder delete left behind (a count in a confirmation is a promise).
- [Frontend](gotchas/frontend.md) — CSP dropping inline styles (Plotly's colorbar, markdown-it's table alignment), CSSOM being exempt from the CSP's protection as well as its rules (so theme colors are strict-hex checked twice), a partial preset that resolves half a theme to `undefined` behind the CSS fallbacks and only says so once it is saved, markdown-it silently refusing `data:image/svg+xml`, `html:false` escaping raw HTML rather than deleting it (so "not rendering" a tag means showing it), why Shiki cannot be used and highlight.js can, Plotly's `<style>` injection and its escape hatch, **Plotly's automargin reserving the MAX of its pushers rather than the sum (so long rotated tick labels were painted straight through the legend, and the agent that hit it hand-truncated its own data to cope)**, **`tickmode: 'array'` — the only way to set custom ticktext — silently turning Plotly's own tick thinning OFF, so a 731-day line drew 731 labels at 0.8px each while the candlestick beside it thinned 400 dates to 40 (plus dates hardcoded onto a category axis, and heatmap cell labels gated on a row COUNT rather than the cell size)**, **a template default change silently relocating every `options` patch written against the old one (container coordinates clamp to 0–1; paper coordinates do not)**, an `<img>`-linked SVG that cannot be themed, WebGL contexts that are never released, a chart vanishing with no error, `splom` drawing nothing on two dimensions, opaque fills, the `options` merge contract, clicks that detach the element they landed on (and the capture phase as the only place a panel that re-renders on click can recognize its own), string-built `<mark>` highlighting, keyboard-opened modals with nowhere to restore focus, a body scroll lock that locks nothing, hover-revealed controls that a touch screen cannot reach (and the copy button that paper never mounts at all), themeRiver dates, swiftshader printing blank 3D charts that look fine on screen, the `.md`-fragment reset that zeroed every margin on paper, `overflow:auto` being a scrollbar on screen and a guillotine in print (a clipped fence loses a line, a clipped table loses whole columns), the sheet-height invariant behind document-mode pagination, rounded table corners that need `border-collapse: separate` and must never become a paper `overflow:hidden` (which would clip columns), a vendored `@font-face` that 403s silently because a CSS `url()` cannot carry the per-request token (and the narrow tokenless exemption that fixes it), responsive `@media` rules that must sit last in the stylesheet or a same-specificity base rule beats them by source order, a stopped-kernel pane that a FAST restart strands forever — a three-miss death gate reused for a state that was already certain, so the WebSocket reconnected underneath it (certainty needs no quorum), and **hiding an element stopping only the work the BROWSER owns** — a hidden tile's lazy `<img>` never fetches, while the off-DOM `<video>` a poster capture builds is beyond the reach of any CSS and went on fetching frames for tiles nobody could see.
- [Testing](gotchas/testing.md) — a green suite that proves nothing, virtual time hiding races, Chrome's `Host`-header trap, clicking an element before its handler is bound, a throwing hook that blames the wrong test (and `readAlive` in a before hook deleting the healthy kernel it could not ping, failing the entire suite at once — 243 tests at the time), tests that cannot fail, an exact assertion on a rendered quantity that transcribed the current layout and thereby PINNED the bug in it (`ticks === 62` was guarding the crammed axis it should have caught), a fixture missing the hard case (which makes a bug *unfailable*), asserting computed values rather than what the stylesheet says, Node 24 subtest socket isolation, in-runner servers invisible to subprocesses (fake kernels must be child processes), a faked `process.platform` splitting workspace identity, shared state-dir coordination, fixed-time waits that break when the suite gets HEAVIER rather than when the code does (mediaui's Space toggle, palette's 6 s save budget) — poll the observable outcome, never the clock — and **negative assertions that are vacuous because the thing could never have happened anyway**: a `[hidden]` check aimed at the one element the UA rule already covers, and a "no fetch" count taken after an earlier step had warmed the cache (pair every negative with the positive control that must fire when the condition flips).
- [Packaging (npm CLI & HappySkills skill)](gotchas/packaging.md) — everything in the skill folder ships (why the runtime moved to npm), the npm `files` allowlist and `npm pack --dry-run`, the size caps solved by relocation rather than shrinking, npx tarball testing (`-p` or it executes the file), a 404 on `npm view` not meaning a name is publishable (the typosquat rule that forced the `@happyskillsai` scope), why the vendored Plotly build is not interchangeable, description validators, mandatory `init` scaffolding, the `.claude/` mirror path, and `npx`'s bare spec running the LOCAL project inside this repo while pinning everyone else to its cache — why every human-pasted command says `@latest`.
